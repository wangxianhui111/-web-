# 服务器项目自动更新方案

## 需求
实现服务器上的项目代码自动更新，当 GitHub 代码更新后，服务器自动拉取最新代码并重启应用。

## 方案选择

### 方案1：手动触发更新（简单可靠）
- 登录服务器后执行更新脚本
- 适合部署频率不高的场景

### 方案2：GitHub Webhooks 自动触发（推荐）
- **触发方式**：GitHub Push → Webhook → 服务器
- **构建方式**：服务器本地构建
- **流程**：
  1. 代码 Push 到 GitHub
  2. GitHub 发送 Webhook 请求到服务器
  3. 服务器收到请求后执行 update.sh
  4. 自动拉取代码、构建 Docker 镜像、重启应用

### 方案3：CI/CD 流水线（完整方案）
- **触发方式**：GitHub Push → GitHub Actions → 服务器
- **构建方式**：GitHub Actions 云端构建 Docker 镜像
- **流程**：
  1. 代码 Push 到 GitHub
  2. GitHub Actions 自动构建 Docker 镜像
  3. 推送到镜像仓库（Docker Hub/阿里云ACR/腾讯云TCR）
  4. 服务器自动拉取新镜像并部署

## 方案对比

| 对比项 | 方案2 Webhook | 方案3 CI/CD |
|--------|---------------|-------------|
| 构建位置 | 服务器本地 | GitHub 云端 |
| 服务器配置 | 需要安装 webhook 工具 | 需要配置 SSH 密钥 |
| 构建速度 | 较慢（服务器配置） | 较快（云端构建） |
| 资源消耗 | 消耗服务器资源 | 消耗 GitHub Actions 资源 |
| 镜像存储 | 无镜像历史 | 镜像仓库存储版本 |
| 复杂度 | 中等 | 较高 |
| 推荐场景 | 个人项目/小型项目 | 生产环境/团队项目 |

## 实施方案

### 步骤1：在服务器上安装 webhook 工具

```bash
# 安装 webhook
sudo apt update
sudo apt install -y webhook

# 或使用 go 安装
go install github.com/adnanh/webhook@latest
```

### 步骤2：创建 webhook 配置文件

创建 `/home/ubuntu/scaffold/webhook.json`：

```json
[
  {
    "id": "redeploy",
    "execute-command": "/home/ubuntu/scaffold/update.sh",
    "command-working-directory": "/home/ubuntu/scaffold"
  }
]
```

### 步骤3：创建更新脚本 update.sh

已在本地创建 `update.sh`，上传到服务器 `/home/ubuntu/scaffold/` 目录。

### 步骤4：启动 webhook 服务

```bash
# 创建 systemd 服务
sudo nano /etc/systemd/system/webhook.service

# 内容：
[Unit]
Description=Webhook for auto deployment
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/webhook -hooks /home/ubuntu/scaffold/webhook.json -port=9000
WorkingDirectory=/home/ubuntu/scaffold
Restart=on-failure

[Install]
WantedBy=multi-user.target

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable webhook
sudo systemctl start webhook
```

### 步骤5：配置安全组

在云服务器安全组中开放端口 9000

### 步骤6：配置 GitHub Webhook

1. 打开 GitHub 仓库页面
2. 进入 Settings → Webhooks
3. 点击 "Add webhook"
4. 配置：
   - Payload URL: `http://你的服务器IP:9000/redeploy`
   - Content type: `application/json`
   - Events: 选择 "Just push events"
5. 点击 "Add webhook"

### 步骤7：测试

1. 本地修改代码并 Push 到 GitHub
2. 查看 GitHub Webhook 是否显示绿色勾
3. 服务器自动执行 update.sh
4. 访问应用验证更新

## 推荐实施顺序

1. 先实现方案1（手动更新脚本）- 立即可用
2. 再实现方案2（Webhook）- 简单自动化
3. 最后实现方案3（CI/CD）- 完整流水线

## 文件清单

| 文件 | 用途 |
|------|------|
| update.sh | 手动更新脚本 |
| webhook.json | Webhook 配置 |
| deploy.sh | 完整部署脚本 |

## 预计工作量
- 方案1：10分钟
- 方案2：30分钟
- 方案3：60分钟
